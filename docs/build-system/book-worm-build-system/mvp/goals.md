# MVP: Goals

- Complete the implementation of all core entity types defined in the Domain Model specification, including relationships, metadata fields and validation rules.
- Implement a public‑facing codex, timeline, atlas and chapter reader that render content from release snapshots with spoiler awareness and role‑based visibility.
- Create a revision history and release system allowing authors to snapshot the current state, publish releases and browse previous editions.
- Build a search engine with weighted indexing of titles, aliases, summaries and tags. Provide filters by type, era, faction, spoiler tier and release.
- Implement a basic continuity engine that detects missing dates, death/appearance contradictions, and out‑of‑order reveal issues.
- Provide full import/export capabilities for Markdown/JSON with front‑matter for all entity types and chapters.
- Ensure the system can be self‑hosted with a documented deployment process and without reliance on third‑party services.
- Deliver documentation for authors on how to use the platform, including admin workflows, release creation and import/export instructions.
