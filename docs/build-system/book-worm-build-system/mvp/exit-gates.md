# MVP: Exit Gates

The MVP may be declared complete when the following conditions are met:

1. **Functionality Coverage:** All features listed in the MVP scope and feature set are implemented and behave according to specification.
2. **Release Readiness:** The release system is stable: releases can be created, published and switched. Public pages always render from the selected release.
3. **Content Integrity:** Import/export round‑trips produce equivalent data structures. No critical data loss occurs during import/export operations.
4. **Search Quality:** Search results are relevant and correct. Hidden and restricted items do not appear in public searches. Filters work correctly.
5. **Continuity Baselines:** The continuity engine detects defined basic issues and the admin UI lists them. Authors can mark issues as resolved or accept exceptions.
6. **Self‑Hosting:** The entire application can be deployed on a clean host using documented steps. Backup and restore scripts work.
7. **Documentation:** Comprehensive author and developer documentation exists and is up to date. A changelog summarises differences from the dev‑use phase.
8. **Testing:** The automated test suite runs in CI and passes. Critical bugs are addressed. Manual QA sign‑off is obtained.
9. **Handoff Preparation:** An issues list and backlog for Beta are compiled. Team leads agree on priorities for the Beta phase.
