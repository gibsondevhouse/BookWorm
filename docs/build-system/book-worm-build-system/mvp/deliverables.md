# MVP: Deliverables

By the end of the MVP phase, the following deliverables should be produced:

- A self‑hosted Book Worm installation that implements all core features described in the PRD: entity management, public codex, timeline, atlas, chapter reader, revision history, release snapshots, search, continuity basics and import/export.
- A complete database schema, API layer and frontend application with polished UI and responsive design.
- A functional release system with the ability to create, publish and browse releases, and switch the active release for public viewing.
- Search functionality with filters and role‑aware/spoiler‑aware indexing. A search UI integrated into the public site.
- A timeline view and atlas with event grouping and map navigation.
- A continuity issue dashboard with recorded issues and resolution workflow.
- Import/export scripts supporting all entity types, chapters and relations, including images. Example data set for demonstration.
- Runtime setup guidance for production‑like deployment and instructions for configuring environment variables and secrets.
- Documentation for authors describing how to manage content, create releases, use spoiler controls, fix continuity issues and run imports/exports.
- Developer documentation updated from the dev‑use stage to reflect new architecture and coding patterns.
- Automated tests covering critical paths with instructions on running them in CI.
