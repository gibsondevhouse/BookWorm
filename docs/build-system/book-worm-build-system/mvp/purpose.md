# MVP: Purpose

The **Minimum Viable Product (MVP)** track aims to transform the dev‑use skeleton into a self‑hosted, end‑to‑end product that satisfies the core functional requirements defined in the PRD. The purpose is to deliver a usable system with essential features—structured entities, public codex, reader, release snapshots, search, timeline, continuity basics and import/export—allowing the author to manage their story world and publish a controlled public view. The MVP is intended for internal use and early adopters rather than general public release.
