# Beta: Exit Gates

The Beta phase can conclude once the following gates are satisfied:

1. **Collaboration Features:** Comments, proposals and review workflows are fully implemented, tested and adopted by the author/editor roles.
2. **Release Inspection:** Users can compare releases and revisions through a diff viewer and annotate releases. The feature is stable.
3. **Search Tuning:** Search returns high‑quality results with fuzzy matching, synonyms and alias support. Filters and spoiler enforcement work correctly.
4. **Expanded Continuity:** New continuity rules detect travel, age, inheritance and magic inconsistencies. Issues can be overridden with justification.
5. **Import/Export Maturity:** Packages with media import/export successfully, conflicts are resolvable and logs are clear. Users can cancel and rollback operations.
6. **Operational Infrastructure:** Logging and metrics are configured. Health endpoints are accessible. Backups run automatically and restore procedures are documented and tested.
7. **Feedback Integration:** Major issues reported by Beta testers are resolved or scheduled. The backlog is trimmed and prioritised for v1.
8. **Documentation:** All features are documented. Onboarding material for editors and operators is up to date.
9. **Handoff Preparedness:** A plan for the v1 phase is produced with scope, priorities and responsibilities defined.
