# Beta: Non‑Goals

- Introducing completely new entity types or major domain changes; the Domain Model is considered stable during Beta.
- Overhauling the design language; only refinements and accessibility improvements are allowed.
- Adding AI‑powered recommendations or automated tagging systems.
- Providing external API access for third‑party integrations; the Beta remains a self‑contained system.
- Expanding to mobile or offline clients; focus remains on the web platform.
- Full performance tuning or scaling for high traffic; this is addressed in v1.
