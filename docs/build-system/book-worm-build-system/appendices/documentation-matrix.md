# Appendix G: Documentation Matrix

| Document                                | Audience     | Maintainer            | When Updated                                   |
|----------------------------------------|--------------|-----------------------|------------------------------------------------|
| Product Requirements Document (PRD)     | Stakeholders | Product Manager       | At start of project; update when goals shift. |
| Domain Model Specification              | Developers   | Lead Architect        | When entity definitions or fields change.      |
| Permissions & Visibility Matrix         | Developers   | Security Lead         | When roles or visibility logic change.         |
| MVP Feature Specification               | Team         | Product Manager       | Before MVP phase; locked at MVP completion.   |
| Implementation Plan / Sequencing        | Team         | Engineering Lead      | At planning; adjust if priorities shift.       |
| Design System Token Spec                | Designers    | Design Lead           | When colours, typography or spacing change.    |
| Search Specification                    | Devs         | Backend Lead          | When indexing rules or ranking change.         |
| Continuity Engine Specification         | Devs         | Narrative Designer    | When new continuity rules are added.           |
| Import/Export Specification             | Devs         | Tooling Engineer      | When formats or behaviours change.             |
| Master Build Prompt for Coding Agent    | Code agent   | Engineering Lead      | At project kickoff; adjust when toolset changes|
| API Contract Specification              | Devs         | Backend Lead          | Whenever endpoints or payloads evolve.         |
| Release/Version Semantics Specification | Devs, PM     | Release Manager       | When release logic changes.                    |
| Build Bible                             | Team         | Engineering Manager   | At each phase; update handoffs and success gates|
| User Guides (Author/Editor/Operator)    | Users        | UX Writer             | Before Beta; refine by v1.                     |
| Developer Handbook                      | Developers   | Engineering Lead      | Continuous; update with patterns and examples. |
