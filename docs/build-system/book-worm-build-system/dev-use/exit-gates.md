# Dev‑Use: Exit Gates

The dev‑use track may be considered complete when the following gates are met:

1. **Environment Reliability:** Any developer can clone the repo, run the provided scripts, and obtain a working stack without manual configuration.
2. **Authentication & Authorisation:** User login and role checks work. Unauthorized users cannot access admin pages.
3. **Core CRUD:** Admin pages enable creation, reading, updating and deletion of Characters, Factions, Locations and Events with server‑side validation.
4. **Design System Adoption:** All admin UI uses components from the design system. Colours, fonts and spacing match the defined tokens.
5. **Release Mechanism:** A basic release can be created and public pages read from it. There is a clear separation between live edits and released content.
6. **Documentation:** The repository includes a README, architecture overview, setup instructions and coding guidelines.
7. **Knowledge Transfer:** The team has reviewed the phase output and identified items for the MVP backlog. Lessons learned are documented.
