# Dev‑Use: Feature Set

1. **Local development environment** — Direct local app runtime with documented setup for the application server and database. Scripts to run and stop the environment.
2. **Basic authentication & roles** — User registration and login; role management with three roles: Public (non‑authenticated), Editor and Author/Admin.
3. **Database schema & migrations** — Initial schema definitions, migration scripts, seed data for users and a few entities.
4. **Design system foundation** — Implementation of typographic scale, spacing units, colour tokens, radii, shadows and motion primitives; core UI components such as buttons, forms, cards and layout primitives.
5. **Core entity CRUD** — Admin pages to create/edit/delete Characters, Factions, Locations and Events. Validation logic for required fields and slugs.
6. **Placeholder public pages** — Very simple public codex index pages to verify the design system with dummy content.
7. **Import/export utilities** — Scripts to parse Markdown with front‑matter into database entities and to export database content back to Markdown.
8. **Documentation** — Initial README with instructions for setup and contributing; guidelines for coding, linting and design tokens.
