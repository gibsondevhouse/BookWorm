# Dev‑Use: Non‑Goals

- Implementing the full set of entity types or complete relational logic; only a subset is needed to validate the architecture.
- Delivering polished or production‑ready public pages; the focus is on internal scaffolding and admin views.
- Implementing release snapshots or versioning; provisional changes may overwrite existing data during this phase.
- Building timeline, atlas, or other discovery-heavy public features; these are deferred to the MVP.
- Creating deep continuity rules or validations beyond basic schema checks.
- Optimising performance or designing for high availability; reliability concerns are addressed later.
- Extensive user documentation or marketing; these items wait until the MVP or Beta phases.
