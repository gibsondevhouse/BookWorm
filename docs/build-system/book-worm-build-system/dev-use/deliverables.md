# Dev‑Use: Deliverables

By the end of the dev‑use phase, the following artifacts should exist:

- A working repository with documented development scripts and local runtime setup.
- A running backend server with authentication and role‑based access control.
- Database schema migrations for users, roles, permissions and core entity tables.
- Seed data for at least one Author/Admin user and a few sample entities.
- A design system package with tokens and core components implemented.
- Admin CRUD pages for Characters, Factions, Locations and Events.
- Basic public index pages demonstrating typographic and layout primitives.
- Initial revision and release tables with a simple release mechanism.
- Import and export scripts that can round‑trip a small dataset.
- Developer documentation covering setup, architecture overview and how to extend the schema.
