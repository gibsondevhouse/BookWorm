# Dev‑Use: Success Criteria

The dev‑use phase is considered successful when:

- Developers can clone the repository, install dependencies, start the app layers locally, and connect required backing services when feature work needs them.
- Basic authentication works: users can sign up, log in and the session persists. Role differentiation is visible in the admin area.
- The database schema is created from migrations, and core tables exist with seed data.
- The design system compiles, and basic components (typography, buttons, cards, forms) display correctly in the admin and placeholder pages.
- Admin CRUD operations for Characters, Factions, Locations and Events are operational, including create, read, update and delete.
- The codebase follows agreed linting and formatting standards and passes unit tests for any utilities implemented.
- Basic import/export scripts can round‑trip a small set of Markdown files for the implemented entities without errors.
- Documentation exists for local setup, code style, and how to add new entity fields and components.
