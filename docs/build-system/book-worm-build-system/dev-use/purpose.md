# Dev‑Use: Purpose

The **dev‑use** track provides a controlled environment for internal developers to establish the scaffolding of Book Worm. It is not intended to deliver user‑visible features but rather to prove that the architecture, coding conventions, schema and design system can support subsequent phases. During this stage the team chooses frameworks, sets up infrastructure, defines core models, and creates the very first CRUD flows. The output of this phase is a working skeleton from which the MVP can be built.
