# Dev‑Use: Scope

Included in this phase:

- Repository bootstrapping: README, license, `.gitignore`, environment examples and local service setup guidance.
- Choosing libraries and frameworks; setting up project configuration (TypeScript config, ESLint/Prettier, testing framework).
- Database initialization and migrations for base tables (users, roles, permissions, entity types, entities, relationships).
- Authentication and simple access control middleware.
- Design system token definitions and core UI kit components.
- Basic admin screens and routes for a few entity types.
- Minimal API routes for authentication and CRUD operations.
- Basic content import/export utilities.
- Minimal tests to verify the environment and first CRUD flows.

Excluded from this phase:

- Public‑facing pages beyond placeholders.
- Timeline, atlas or advanced release features.
- Continuity engine or complex validations.
- Advanced file upload, map rendering or graph visualisation.
- Real production deployment configuration or high‑availability concerns.
