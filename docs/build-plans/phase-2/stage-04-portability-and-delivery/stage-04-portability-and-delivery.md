# Stage 04: Portability and Delivery

## Purpose

Close the MVP portability and operational gaps so Book Worm is inspectable across releases, movable in and out of files, and startable by another operator.

## Parts

1. Release History and Archive Browsing
2. Import Export and Self-Hosting Baseline
3. Phase 2 Verification Gate

## Outputs

- release-history and archive inspection behavior
- import and export paths for the implemented MVP scope
- self-hosting guidance and a verification gate for the full phase

## Exit Criteria

- authors can inspect prior releases without mutating them
- implemented content can be exported and re-imported through documented flows
- another developer can start and validate the implemented MVP slice from repository documentation
