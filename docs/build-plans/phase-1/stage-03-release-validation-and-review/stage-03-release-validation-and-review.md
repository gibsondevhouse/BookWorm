# Stage 03: Release Validation and Review

## Purpose

Strengthen the release workflow so activation depends on explicit validation and review behavior rather than on optimistic release composition.

## Parts

1. Release Validation Baseline
2. Release Review Surface
3. Activation Guardrails

## Outputs

- dependency-aware release validation
- review-oriented release summary behavior
- activation rules that block incomplete or invalid releases
- clearer release-readiness feedback for admin users

## Exit Criteria

- releases can be checked for readiness before activation
- admins can inspect what a release includes and what it still lacks
- activation is blocked when validation fails
