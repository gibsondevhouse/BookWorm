# Stage 01: Expanded Content Spine

## Purpose

Extend the Phase 0 schema and repository model so Book Worm can support more than one representative entity and can begin tracking relationship revisions explicitly.

## Parts

1. Additional Entity Modeling
2. Relationship Revision Foundation
3. Dependency Resolution Baseline

## Outputs

- expanded schema for the next supported content types
- repository and persistence updates for the broader entity model
- first relationship revision storage path
- dependency resolution rules that release validation can build on

## Exit Criteria

- new supported content types can be stored and versioned
- relationships are modeled as independently revisioned records
- dependency metadata needed for release validation is persisted and queryable
