# Phase 1

## Purpose

Phase 1 turns the Phase 0 proof slice into a broader production baseline. Its job is to expand the content model, authoring coverage, and release validation path so Book Worm can move beyond a single representative entity.

## Outcome

At the end of Phase 1, the project should have:

- broader entity coverage beyond the first Character slice
- relationship revision handling aligned with the resolved build decisions
- release validation that checks dependency completeness before activation
- expanded admin authoring flows for the newly supported content types
- broader public read coverage derived only from the active release

## Scope

Phase 1 includes the minimum work needed to move from a validated prototype slice to a usable multi-entity authoring baseline.

### Included

- additional entity modeling required for broader canon coverage
- relationship versioning foundation
- admin retrieval and editing for the newly supported content types
- release validation and review support for dependency-complete publication
- expanded public read paths and discovery scaffolding
- test and verification coverage for the broader slice

### Excluded

- final search engine selection and production indexing infrastructure
- object storage implementation beyond current placeholders
- import and export workflows beyond planning stubs
- proposal, comments, and collaboration workflow surfaces
- advanced spoiler and field-level shaping beyond the minimum needed for the phase

## Stage Breakdown

1. Stage 01: Expanded Content Spine
2. Stage 02: Authoring Surface Expansion
3. Stage 03: Release Validation and Review
4. Stage 04: Public Coverage and Verification

## Exit Criteria

- additional supported entity types can be drafted and persisted with revisions
- relationships can be versioned and included in release composition
- release activation is blocked when required dependencies are missing
- public reads can resolve the newly supported content through the active release
- verification artifacts demonstrate that broader multi-entity flows still preserve draft and release separation
- phase tracker and part documents are updated to reflect completion state
