# Stage 03: Data Model and Release Spine

## Purpose

Build the minimum schema and release composition model required to prove that Book Worm can separate draft authoring from public canon.

## Parts

1. Prisma Initial Schema
2. Revision and Release Composition
3. Seed and Fixture Baseline

## Outputs

- initial Prisma schema
- first migrations
- minimal versioned content model
- release composition tables and access patterns
- seed data and fixtures for early testing

## Exit Criteria

- schema supports users, sessions, entities, revisions, releases, and release entries
- migrations apply cleanly
- seed data can create a usable initial environment
