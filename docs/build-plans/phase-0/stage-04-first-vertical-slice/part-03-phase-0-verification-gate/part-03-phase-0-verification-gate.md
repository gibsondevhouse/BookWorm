# Part 03: Phase 0 Verification Gate

## Objective

Validate that the first vertical slice proves the architecture rather than only demonstrating isolated features.

## Work To Be Done

- verify local stack startup from documented commands
- verify session creation and protected access behavior
- verify draft save creates a revision
- verify release inclusion and activation behavior
- verify public read resolves only from the active release
- verify unreleased changes remain hidden
- record any architecture-level failures discovered during the slice

## Deliverables

- Phase 0 verification checklist
- recorded pass or fail status for the core architectural assertions

## Dependencies

- all prior Phase 0 parts complete

## Completion Check

- the phase exit criteria in [phase-0.md](../../phase-0.md) are demonstrably satisfied
- the project is ready to enter the next implementation phase without unresolved foundation ambiguity
