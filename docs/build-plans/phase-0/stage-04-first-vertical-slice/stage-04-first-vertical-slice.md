# Stage 04: First Vertical Slice

## Purpose
Prove the chosen architecture with one end-to-end content flow before broader feature expansion begins.

## Parts
1. Admin Character Draft Flow
2. Active Release Public Read
3. Phase 0 Verification Gate

## Outputs
- first admin write path
- first release-bound public read path
- tests or manual verification artifacts demonstrating draft/public separation

## Exit Criteria
- an authenticated author can create or update a representative character draft
- the change creates a revision
- a release can include that revision and become active
- the public surface reads only from the active release
- unreleased edits remain hidden from the public surface
